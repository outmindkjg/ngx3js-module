* A global instance of the [LoadingManager](https://outmindkjg.github.io/ngx3js-doc/#/docs/ngxapi/en/LoadingManager), used by most loaders when no custom manager has been specified. This will be sufficient for most purposes, however there may be times when you desire separate loading managers for say, textures and models. * ### Code Example * ```javascript * You can optionally set the [onStart](https://outmindkjg.github.io/ngx3js-doc/#/docs/ngxapi/en/LoadingManager.onStart), [onLoad](https://outmindkjg.github.io/ngx3js-doc/#/docs/ngxapi/en/LoadingManager.onLoad), * [onProgress](https://outmindkjg.github.io/ngx3js-doc/#/docs/ngxapi/en/LoadingManager.onProgress), [onError](https://outmindkjg.github.io/ngx3js-doc/#/docs/ngxapi/en/LoadingManager.onStart) functions for the manager. * These will then apply to any loaders using the DefaultLoadingManager. * Note that these shouldn't be confused with the similarly named functions of individual loaders, as they are intended for displaying information about the overall status of loading, rather than dealing with the data that has been loaded. * THREE.DefaultLoadingManager.onStart = function ( url, itemsLoaded, itemsTotal ) { * console.log( 'Started loading file: ' + url + '.\nLoaded ' + itemsLoaded + ' of ' + itemsTotal + ' files.' ); }; * THREE.DefaultLoadingManager.onLoad = function ( ) { * console.log( 'Loading Complete!'); }; * THREE.DefaultLoadingManager.onProgress = function ( url, itemsLoaded, itemsTotal ) { * console.log( 'Loading file: ' + url + '.\nLoaded ' + itemsLoaded + ' of ' + itemsTotal + ' files.' ); }; * THREE.DefaultLoadingManager.onError = function ( url ) { * console.log( 'There was an error loading ' + url ); };
